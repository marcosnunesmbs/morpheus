# Analyze Command Documentation

## Command
`speckit.analyze`

## Purpose
Analyzes existing features, code, or specifications to understand current state and identify improvements.

## When to Use
- Before planning a new feature that interacts with existing code
- During refactoring
- For code quality assessment
- To understand legacy features

## Prerequisites
- Code or specifications to analyze
- Clear analysis objectives

## Execution Flow

1. **Define Scope**: Identify what needs analysis
2. **Gather Context**: Read relevant files and docs
3. **Analyze**: Evaluate against criteria
4. **Report**: Document findings
5. **Recommend**: Suggest improvements or next steps

## Analysis Types

- **Code Quality**: Structure, patterns, maintainability
- **Feature Gap**: Missing functionality
- **Specification Compliance**: Code vs spec alignment
- **Technical Debt**: Issues requiring refactoring
- **Dependencies**: Integration points and impacts

## Output

Analysis report with:
- Current state assessment
- Issues identified
- Recommendations
- Impact analysis

## Agent-Specific Guidance

This command is agent-agnostic and can be executed by any AI assistant with code analysis capabilities.
