# Clarify Command Documentation

## Command
`speckit.clarify`

## Purpose
Helps clarify ambiguous requirements or resolve "NEEDS CLARIFICATION" markers in specifications and plans.

## When to Use
- When encountering unclear requirements
- Before implementation of ambiguous features
- To resolve NEEDS CLARIFICATION markers

## Prerequisites
- Feature specification with unclear sections
- Context about the ambiguity

## Execution Flow

1. **Identify Ambiguity**: Locate unclear sections
2. **Gather Context**: Review related documentation
3. **Ask Questions**: Present clarification questions to stakeholders
4. **Update Docs**: Replace NEEDS CLARIFICATION with concrete details
5. **Validate**: Ensure clarity is achieved

## Common Clarifications

- Technical implementation details
- Non-functional requirements
- Edge case handling
- Integration points
- Performance criteria

## Output

- Updated specification with resolved ambiguities
- Decision log
- Rationale for choices made

## Agent-Specific Guidance

This command is agent-agnostic and can be executed by any AI assistant.
