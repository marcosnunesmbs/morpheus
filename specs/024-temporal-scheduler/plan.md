# Implementation Plan: Chronos — Temporal Intent Engine

**Branch**: `024-temporal-scheduler` | **Date**: 2026-02-21 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/024-temporal-scheduler/spec.md`

## Summary

Chronos is a local-first temporal scheduler that persists user-defined prompt jobs and triggers the Oracle at the right time. It introduces two new SQLite tables (`chronos_jobs`, `chronos_executions`) to `short-memory.db`, a polling `ChronosWorker` (60s interval, isRunning guard), a full REST API at `/api/chronos`, a new **Chronos** dashboard page, global settings in Zaion, and Telegram command support (`/chronos`, `/chronos_list`, etc.). Natural language date parsing uses `chrono-node`; cron expression handling uses `cron-parser` + `cronstrue`; timezone arithmetic uses `date-fns-tz`.

**Target Platform**: Local Node.js daemon (same process as Oracle/TaskWorker)

**Constraints**: Minimum 60s check interval; no duplicate job triggers; full restart resilience
**Scale/Scope**: Single-user local deployment; max 100 concurrent active jobs (configurable)

## Technical Design Decisions

### ChronosWorker Loop

```typescript
// src/runtime/chronos/worker.ts (pseudocode)
class ChronosWorker {
  private timer: NodeJS.Timeout | null = null;
  private isRunning = false;

  start(): void {
    if (this.timer) return;
    this.timer = setInterval(() => void this.tick(), this.pollIntervalMs);
  }

  private async tick(): Promise<void> {
    if (this.isRunning) return;
    this.isRunning = true;
    try {
      const dueJobs = this.repo.getDueJobs(Date.now()); // enabled=1, next_run_at <= now
      for (const job of dueJobs) {
        void this.executeJob(job); // fire-and-forget per job; tracked via executions table
      }
    } finally {
      this.isRunning = false;
    }
  }

  private async executeJob(job: ChronosJob): Promise<void> {
    const execId = crypto.randomUUID();
    this.repo.insertExecution({ id: execId, job_id: job.id, status: 'running', ... });
    try {
      await oracle.chat(job.prompt, `chronos-job-${job.id}`);
      this.repo.completeExecution(execId, 'success');
    } catch (err) {
      this.repo.completeExecution(execId, 'failed', err.message);
    } finally {
      if (job.schedule_type === 'once') {
        this.repo.disableJob(job.id);
      } else {
        const next = parseNextRun(job.cron_normalized!, job.timezone);
        this.repo.updateNextRun(job.id, next);
      }
      this.repo.pruneExecutions(job.id, 100); // keep last 100
    }
  }
}
```

### Schedule Parser

```typescript
// src/runtime/chronos/parser.ts (pseudocode)
function parseScheduleExpression(
  expression: string,
  type: ScheduleType,
  opts: ParseScheduleOptions
): ParsedSchedule {
  switch (type) {
    case 'once':
      // chrono-node with timezone-aware reference date
      const parsed = chrono.parseDate(expression, { timezone: opts.timezone });
      return { type, next_run_at: parsed.getTime(), cron_normalized: null, human_readable: formatDate(parsed) };

    case 'cron':
      // validate with cron-parser; human_readable via cronstrue
      const interval = CronParser.parseExpression(expression, { tz: opts.timezone });
      return { type, next_run_at: interval.next().getTime(), cron_normalized: expression, human_readable: cronstrue.toString(expression) };

    case 'interval':
      // Convert "every 30 minutes" → cron "*/30 * * * *"
      const cron = intervalToCron(expression);
      const next = CronParser.parseExpression(cron, { tz: opts.timezone }).next().getTime();
      return { type, next_run_at: next, cron_normalized: cron, human_readable: cronstrue.toString(cron) };
  }
}
```

### Telegram Commands

| Command | Handler | Behavior |
|---------|---------|---------|
| `/chronos <prompt + time>` | `handleChronosCreate` | Parse intent, confirm with user, persist on confirmation |
| `/chronos_list` | `handleChronosList` | List active jobs (ID + summary + next_run) |
| `/chronos_view <id>` | `handleChronosView` | Show single job details + last execution |
| `/chronos_disable <id>` | `handleChronosDisable` | Disable job, confirm |
| `/chronos_enable <id>` | `handleChronosEnable` | Enable job + show new next_run, confirm |
| `/chronos_delete <id>` | `handleChronosDelete` | Delete job, confirm |

Confirmation flow for `/chronos create`: Two-message pattern (same as Trinity's interactive flows):
1. Chronos parses the expression → replies with interpretation + asks "Confirm? (yes/no)"
2. User replies "yes" → job saved, confirmation with job ID
